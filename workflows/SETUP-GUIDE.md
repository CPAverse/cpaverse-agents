# CPAverse Agent — n8n Workflow Setup Guide

## Prerequisites

Before importing workflows into n8n Cloud, you need these credentials configured:

### 1. Claude API (Anthropic)
- **Type:** HTTP Header Auth
- **Header Name:** `x-api-key`
- **Header Value:** Your Anthropic API key
- **Get key at:** https://console.anthropic.com/settings/keys

### 2. GitHub Personal Access Token
- **Type:** HTTP Header Auth
- **Header Name:** `Authorization`
- **Header Value:** `Bearer ghp_YOUR_TOKEN_HERE`
- **Create at:** https://github.com/settings/tokens
- **Scopes needed:** `repo` (full control of private repositories)

### 3. Gmail OAuth2
- **Type:** Gmail OAuth2 (built-in n8n node)
- **Account:** josh@joshmauercpa.com
- **Setup:** n8n will guide you through Google OAuth flow

### 4. Firecrawl API (Phase 4)
- **Type:** HTTP Header Auth
- **Header Name:** `Authorization`
- **Header Value:** `Bearer fc-YOUR_FIRECRAWL_KEY`
- **Your key prefix:** fc-51ee2...

## Workflow 1: Morning Report

**File:** `morning-report-workflow.json`
**n8n Workflow ID:** `FkPhsXWowDd0pna2` (already imported)

### What it does:
1. Fires daily at 7:00 AM Central Time
2. Reads the activity log from GitHub (cpaverse-agents/logs/daily-activity.md)
3. Sends log content to Claude API to format the morning report
4. Parses the report into subject line and HTML body
5. Sends the report email to josh@joshmauercpa.com
6. Archives the day's log to GitHub (logs/archive/YYYY-MM-DD.md)
7. Resets the daily activity log for the new day

### Setup steps:
1. In n8n, go to Settings > General > Timezone and set to **America/Chicago**
2. Import the workflow JSON file
3. Open each node and replace placeholder values:
   - `{GITHUB_OWNER}` → your GitHub username
   - `GMAIL_CREDENTIAL_ID` → select your Gmail credential
   - Claude API credential → select your Anthropic HTTP Header Auth credential
   - GitHub credential → select your GitHub HTTP Header Auth credential
4. Activate the workflow
5. Test by clicking "Execute Workflow" manually

## Workflow 2: TaxDome Monitor

**File:** `taxdome-monitor-workflow.json`
**n8n Workflow ID:** `OgaHAAHFLRlpprAW` (already imported)

### What it does:
1. Listens for webhook events from TaxDome
2. Classifies events (document upload, client message, job status change, deadline, request response)
3. Routes to appropriate processing branch
4. Uses Claude API to analyze each event and determine next action
5. Logs all activity to GitHub
6. Sends urgent alerts immediately via email

### Setup steps:
1. Import the workflow JSON file
2. Replace all placeholder values (same as Workflow 1)
3. Activate the workflow — this will generate a webhook URL
4. Copy the webhook URL from the TaxDome Webhook node
5. In TaxDome: go to Settings > Integrations > Webhooks
6. Add the n8n webhook URL
7. Select events to monitor: document uploads, messages, job updates, deadlines
8. Test by uploading a test document in TaxDome

### TaxDome Webhook Configuration:
- **URL:** (generated by n8n when workflow is activated)
- **Method:** POST
- **Events:** All available events
- **Note:** TaxDome's webhook payload structure may vary. The classifier node handles common formats, but may need adjustment based on TaxDome's actual API response format.

## Architecture Note

These n8n Cloud workflows run 24/7 on n8n's servers — no laptop required. The Claude API is called server-to-server for intelligence. Gmail and GitHub are accessed via their APIs. This is the production runtime for the CPAverse Tax Prep Agent.

A backup Cowork scheduled task also exists for the Morning Report and can be used when testing locally or as a failover.
